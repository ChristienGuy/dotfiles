# zsh config

Portable zsh config using [antidote](https://antidote.sh/) (plugin manager) and [git-spice](https://abhinav.github.io/git-spice/) (`gs`), plus core shell settings and aliases.

## Repo structure

- **`.zsh_core`** — Main config: history, Starship, fzf, zoxide, antidote, git-spice completion, `grb-sign`, and optional NVM/pyenv/PHP paths. Sources `.zsh_aliases` and `.zsh_functions`.
- **`.zsh_aliases`** — Aliases (e.g. `cod`/`curs`, git shortcuts, git branch helpers). Edit this file to add or change aliases.
- **`.zsh_functions`** — Shell functions (e.g. `fsb`, `fsbd` for fzf-based git branch switch/delete). Edit this file to add or change functions.
- **`.zsh_plugins.txt`** — Antidote plugin list (zephyr completion/compstyle, npm scripts autocomplete, zsh-users syntax/autosuggestions/history-substring-search). Edit this to add or remove plugins.
- **`.zsh_plugins.zsh`** — Generated by antidote; do not edit. Created when you run `antidote load` or open a new shell after installing antidote.

## How to use

1. Clone or symlink this repo to `~/.zsh` (the config expects `~/.zsh/.zsh_plugins.txt`).
2. In your `~/.zshrc`, source the core file:

```bash
[ -f ~/.zsh/.zsh_core ] && source ~/.zsh/.zsh_core
```

## Prerequisites

These tools are used by the config and must be installed separately. See each project's install docs:

| Tool | Purpose | Install |
|------|---------|--------|
| **Antidote** | Zsh plugin manager; loads `.zsh_plugins.txt` | [antidote.sh/install](https://antidote.sh/install) — e.g. `brew install antidote` |
| **git-spice** | Shell completion (`gs`); stacked branches | [git-spice install](https://abhinav.github.io/git-spice/start/install/) — e.g. `brew install git-spice` |
| **Starship** | Shell prompt | [starship.rs](https://starship.rs/) — e.g. `brew install starship` |
| **fzf** | Fuzzy finder; used by `fsb` branch picker | [fzf installation](https://github.com/junegunn/fzf#installation) — e.g. `brew install fzf` |
| **zoxide** | Smart directory jumper (`z`, `zi`) | [zoxide](https://github.com/ajeetdsouza/zoxide) — e.g. `brew install zoxide` |
| **nvm** | Node version manager (optional) | [nvm install](https://github.com/nvm-sh/nvm#installing-and-updating) — run the install script, then ensure your shell sources this config so nvm loads |

After installing antidote, opening a new shell (or running `antidote load ~/.zsh/.zsh_plugins.txt`) will clone plugins and generate `.zsh_plugins.zsh`.
